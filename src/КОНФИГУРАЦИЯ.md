# Конфигурация проекта

Этот файл содержит важную информацию о настройках проекта для Ubuntu 22 сервера.

## 📦 Зависимости

### Node.js
- **Рекомендуемая версия**: Node.js 20.x LTS
- **Минимальная версия**: Node.js 18.x
- **npm версия**: 10.x

### Пакеты проекта
Все зависимости указаны в `package.json`:
- React 18.2.0
- TypeScript 5.3.3
- Vite 5.1.0
- Tailwind CSS 4.0.0
- И другие...

## 🔧 Основные файлы конфигурации

### vite.config.ts
Настройки сборщика Vite:
- Порт по умолчанию: **3000**
- Host: `true` (доступен извне)
- Output директория: `dist/`
- TypeScript target: ES2020

### package.json
Скрипты для запуска:
- `npm run dev` - режим разработки
- `npm run build` - сборка проекта
- `npm run preview` - предпросмотр production сборки

### tsconfig.json
Настройки TypeScript компилятора

### postcss.config.js
Настройки PostCSS для Tailwind CSS

## 🌐 Supabase настройки

### Файл: utils/supabase/info.tsx
Содержит:
- `projectId` - ID вашего Supabase проекта
- `publicAnonKey` - публичный анонимный ключ

**Важно**: Эти данные уже настроены в проекте. Если вам нужно использовать свой Supabase проект, измените эти значения.

### Где найти данные:
1. Откройте ваш проект в [Supabase Dashboard](https://supabase.com)
2. Settings → API
3. Скопируйте:
   - Project URL (из него возьмите project ID)
   - anon/public key

## 🖥️ Системные требования

### Ubuntu 22.04
- **ОС**: Ubuntu 22.04 LTS (или новее)
- **RAM**: минимум 1GB (рекомендуется 2GB+)
- **Диск**: минимум 2GB свободного места
- **Порт**: 3000 (или любой другой) должен быть открыт

### Firewall
Откройте необходимые порты:
```bash
sudo ufw allow 3000/tcp    # Для приложения
sudo ufw allow 80/tcp      # Для HTTP (если используете Nginx)
sudo ufw allow 443/tcp     # Для HTTPS (если используете Nginx)
```

## ⚙️ Рекомендации для production

### 1. Используйте systemd
Создайте service для автозапуска (см. UBUNTU_INSTALL.md)

### 2. Используйте Nginx
Для SSL и reverse proxy (опционально)

### 3. Мониторинг
Следите за логами:
```bash
sudo journalctl -u url-shortener -f
```

### 4. Обновления
Регулярно обновляйте зависимости:
```bash
npm update
npm audit fix
```

## 🔄 Обновление проекта

### Если используете Git
```bash
git pull
npm install
npm run build
sudo systemctl restart url-shortener
```

### Если используете ZIP
1. Скачайте новую версию
2. Сохраните `utils/supabase/info.tsx`
3. Распакуйте новые файлы
4. Восстановите `utils/supabase/info.tsx`
5. Выполните:
```bash
npm install
npm run build
sudo systemctl restart url-shortener
```

## 📊 Производительность

### Режим разработки (dev)
- Hot reload включен
- Source maps включены
