# ✅ Чеклист развертывания на Ubuntu 22

Используйте этот чеклист для проверки готовности проекта к развертыванию.

## 📥 Перед началом

- [ ] У вас есть Ubuntu 22.04 сервер с SSH доступом
- [ ] Открыт порт 3000 (или другой по выбору) на сервере
- [ ] У вас есть Supabase аккаунт и проект
- [ ] Проект скачан на сервер (через Git или ZIP)

## 🔧 Базовая установка

- [ ] Node.js 20.x установлен (`node --version`)
- [ ] npm установлен (`npm --version`)
- [ ] Зависимости установлены (`npm install` выполнен)
- [ ] Проект собран (`npm run build` выполнен)

## ⚙️ Конфигурация

- [ ] Supabase настройки обновлены в `utils/supabase/info.tsx`
  - [ ] `projectId` заменен на ваш
  - [ ] `publicAnonKey` заменен на ваш
- [ ] Порт настроен (если не 3000)
- [ ] Firewall настроен (порт открыт)

## 🧪 Тестирование

- [ ] Приложение запускается локально
  ```bash
  npm run preview -- --host 0.0.0.0 --port 3000
  ```
- [ ] Сайт доступен по адресу `http://localhost:3000`
- [ ] Сайт доступен по IP сервера извне
- [ ] Можно создать короткую ссылку
- [ ] Редирект по короткой ссылке работает
- [ ] Регистрация работает (если настроена)

## 🔄 Автозапуск (systemd)

- [ ] Service файл создан `/etc/systemd/system/url-shortener.service`
- [ ] Пути в service файле корректны
- [ ] Пользователь в service файле корректен
- [ ] Service включен (`sudo systemctl enable url-shortener`)
- [ ] Service запущен (`sudo systemctl start url-shortener`)
- [ ] Service работает (`sudo systemctl status url-shortener`)
- [ ] Логи читаются (`sudo journalctl -u url-shortener -f`)

## 🌐 Nginx (опционально)

Если используете Nginx:

- [ ] Nginx установлен
- [ ] Конфигурация создана в `/etc/nginx/sites-available/`
- [ ] Symbolic link создан в `/etc/nginx/sites-enabled/`
- [ ] Конфигурация проверена (`sudo nginx -t`)
- [ ] Nginx перезапущен (`sudo systemctl restart nginx`)
- [ ] Сайт доступен через домен

## 🔐 SSL (опционально)

## 🔥 Безопасность

## 🔧 Быстрые команды для проверки

```bash
# Проверка статуса service
sudo systemctl status url-shortener

# Проверка логов
sudo journalctl -u url-shortener -f

# Проверка доступности
curl http://localhost:3000

# Проверка порта
sudo netstat -tulpn | grep 3000

# Проверка процессов
ps aux | grep node

# Перезапуск
sudo systemctl restart url-shortener
```

## ❗ Если что-то не работает

1. Проверьте логи: `sudo journalctl -u url-shortener -f`
2. Проверьте статус: `sudo systemctl status url-shortener`
3. Проверьте конфигурацию: `utils/supabase/info.tsx`
4. Проверьте firewall: `sudo ufw status`
5. Проверьте порт: `sudo lsof -i :3000`

См. раздел "Решение проблем" в `UBUNTU_INSTALL.md`
