# Руководство по управлению контентом

## Обзор

Теперь ваш сайт поддерживает динамическое управление контентом через Supabase. Это означает, что вы можете редактировать любые тексты на сайте через админ-панель без необходимости пересборки и перезагрузки приложения.

## Как это работает

1. **Таблица content в Supabase** - хранит все тексты сайта
2. **ContentProvider** - загружает контент при запуске приложения и предоставляет его всем компонентам
3. **ContentManager** - компонент для редактирования текстов в админ-панели
4. **Fallback контент** - если таблица не настроена, используются текста по умолчанию из кода

## Первоначальная настройка

### Шаг 1: Создание таблицы в Supabase

1. Откройте ваш проект в Supabase Dashboard
2. Перейдите в раздел **SQL Editor**
3. Скопируйте и выполните SQL-скрипт из файла `CONTENT_SETUP.md` (раздел "Создание таблицы")
4. Выполните SQL для добавления начальных данных (раздел "Добавление начальных данных")

### Шаг 2: Настройка прав администратора

В SQL-скрипте найдите строку:
```sql
auth.jwt() ->> 'email' = 'admin@yourdomain.com'
```

Замените `'admin@yourdomain.com'` на email вашего администратора.

### Шаг 3: Проверка в Header.tsx

Откройте файл `/components/Header.tsx` и найдите строку:
```typescript
const isAdmin = user?.email === 'admin@yourdomain.com';
```

Замените `'admin@yourdomain.com'` на тот же email администратора.

## Использование системы управления контентом

### Доступ к админ-панели

1. Войдите на сайт под учетной записью администратора
2. В хедере появится кнопка **"Админ"**
3. Нажмите на нее, чтобы открыть административную панель

### Редактирование контента

1. В админ-панели перейдите на вкладку **"Управление контентом"**
2. Используйте поиск или фильтры по категориям для поиска нужного текста
3. Нажмите кнопку **"Редактировать"** рядом с нужным текстом
4. Измените текст в поле ввода
5. Нажмите **"Сохранить"**

Изменения вступают в силу **немедленно**! Все пользователи увидят новые тексты при следующей загрузке страницы.

### Структура контента

Каждый текстовый элемент имеет:

- **Ключ** (key) - уникальный идентификатор, например `home.hero.title`
- **Значение** (value) - сам текст, который отображается на сайте
- **Категория** (category) - группировка текстов (home, auth, dashboard, admin, header)
- **Описание** (description) - пояснение, где этот текст используется

### Категории контента

- **home** - главная страница (hero секция, преимущества, CTA и т.д.)
- **header** - шапка сайта (логотип, кнопки)
- **auth** - модальное окно авторизации
- **dashboard** - личный кабинет пользователя
- **admin** - административная панель

## Добавление новых текстов

Если вам нужно добавить новый редактируемый текст:

### Через Supabase Dashboard

1. Откройте **Table Editor** → **content**
2. Нажмите **Insert row**
3. Заполните поля:
   - `key` - уникальный ключ (например, `home.new.text`)
   - `value` - текст
   - `category` - категория
   - `description` - описание
4. Сохраните

### Через SQL

```sql
INSERT INTO content (key, value, category, description) 
VALUES (
  'home.new.text',
  'Новый текст',
  'home',
  'Описание где используется'
);
```

### В коде компонента

Используйте хук `useText`:

```tsx
import { useText } from '../utils/content';

function MyComponent() {
  const myText = useText('home.new.text', 'Fallback текст');
  
  return <div>{myText}</div>;
}
```

## Резервное копирование

**Важно:** Регулярно делайте резервные копии таблицы `content`.

### Экспорт через Supabase

1. Откройте **Table Editor** → **content**
2. Нажмите на кнопку **Export** (три точки в правом верхнем углу)
3. Выберите формат **CSV** или **JSON**
4. Сохраните файл

### Экспорт через SQL

```sql
COPY (SELECT * FROM content) 
TO '/tmp/content_backup.csv' 
WITH (FORMAT CSV, HEADER);
```

## Восстановление

Если что-то пошло не так, вы всегда можете:

1. Восстановить данные из резервной копии
2. Повторно выполнить SQL-скрипт с начальными данными
3. Приложение продолжит работать с fallback текстами из кода

## Мониторинг изменений

В админ-панели для каждого текста отображается:
- Дата последнего обновления
- Текущее значение

Рекомендуется вести журнал важных изменений контента отдельно.

## Безопасность

- ✅ Только пользователи с правами администратора могут редактировать контент
- ✅ Все пользователи могут читать контент (для отображения на сайте)
- ✅ Изменения логируются через поле `updated_at`
- ⚠️ Убедитесь, что email администратора правильно указан в RLS политиках

## Производительность

- Контент загружается один раз при запуске приложения
- Кэшируется в памяти браузера
- Обновляется автоматически при редактировании в админ-панели
- Минимальная нагрузка на Supabase

## Поддержка

Если у вас возникли проблемы:

1. Проверьте, что таблица `content` создана в Supabase
2. Убедитесь, что RLS политики настроены правильно
3. Проверьте email администратора в настройках
4. Посмотрите консоль браузера на наличие ошибок

## Полезные SQL-запросы

### Посмотреть все тексты определенной категории
```sql
SELECT * FROM content WHERE category = 'home' ORDER BY key;
```

### Найти текст по ключевому слову
```sql
SELECT * FROM content WHERE value ILIKE '%маркетинг%';
```

### Обновить несколько текстов одновременно
```sql
UPDATE content 
SET value = 'Новый текст' 
WHERE category = 'home' AND key LIKE 'home.hero%';
```

### Посмотреть последние изменения
```sql
SELECT key, value, updated_at 
FROM content 
ORDER BY updated_at DESC 
LIMIT 10;
```
