# 🚀 Установка на Ubuntu 22.04 (Порт 3000)

Быстрая инструкция для запуска URL Shortener на вашем сервере Ubuntu 22 на порту 3000.

---

## ⚡ Автоматическая установка (рекомендуется)

Самый быстрый способ - использовать автоматический скрипт:

```bash
# Скачайте скрипт
wget https://raw.githubusercontent.com/ваш-репозиторий/url-shortener/main/quick-deploy.sh

# Или если файлы уже на сервере:
cd /путь/к/проекту
chmod +x quick-deploy.sh
bash quick-deploy.sh
```

Скрипт автоматически:
- Установит Node.js 20 (если нужно)
- Настроит Supabase конфигурацию
- Установит зависимости
- Соберет проект
- Установит и настроит PM2
- Запустит приложение на порту 3000

⏱️ Время: 5-10 минут

**Или следуйте ручной установке ниже** ⬇️


## 📦 Шаг 2: Установка Node.js 20

```bash
# Обновляем систему
sudo apt update && sudo apt upgrade -y

# Устанавливаем Node.js 20.x
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# Проверяем версию
node --version  # Должно быть v20.x.x

```

## 📁 Шаг 3: Копирование проекта на сервер

### Вариант А: Если файлы на вашем локальном компьютере

```bash
# На локальном компьютере, в папке с проектом:
```


### Вариант В: Создать папку и загрузить файлы вручную

```bash
# На сервере:
mkdir -p ~/url-shortener

# Затем используйте SFTP или scp для загрузки файлов
```

## 🔧 Шаг 4: Настройка Supabase

### 4.1 Получите данные из Supabase Dashboard

1. Откройте [app.supabase.com](https://app.supabase.com)
2. Выберите ваш проект
3. Перейдите в **Settings → API**
4. Скопируйте:
   - **Project URL** (например: `https://xxxxxxxxxxxxx.supabase.co`)
   - **Project ID** (например: `xxxxxxxxxxxxx` - часть URL без https:// и .supabase.co)
   - **anon public key** (длинный токен)

### 4.2 Обновите файл конфигурации

```bash
cd ~/url-shortener

# Откройте файл для редактирования
nano utils/supabase/info.tsx
```

Замените содержимое на:

```typescript
/* AUTOGENERATED FILE - DO NOT EDIT CONTENTS */

export const projectId = "ваш-project-id"
export const publicAnonKey = "ваш-anon-key"
```

Сохраните: `Ctrl+O`, затем `Enter`, затем `Ctrl+X`

## 🚀 Шаг 7: Запуск на порту 3000

### Вариант А: Простой запуск для тестирования

```bash
npm run preview -- --host 0.0.0.0 --port 3000
```

Сайт будет доступен по адресу: `http://ваш-ip:3000`

⚠️ **Важно:** Этот способ работает только пока открыта SSH сессия.

### Вариант Б: Запуск в фоне с помощью PM2 (рекомендуется)

```bash
# Установка PM2
sudo npm install -g pm2

# Запуск приложения
pm2 start npm --name "url-shortener" -- run preview -- --host 0.0.0.0 --port 3000

# Настройка автозапуска при перезагрузке сервера
pm2 startup
pm2 save

# Полезные команды PM2:
pm2 status              # Статус приложения
pm2 logs url-shortener  # Просмотр логов
pm2 restart url-shortener  # Перезапуск
pm2 stop url-shortener     # Остановка
```

### Вариант В: Использование screen (если нет PM2)

```bash
# Установка screen
sudo apt install screen -y

# Создание новой сессии
screen -S urlshortener

# Запуск приложения
npm run preview -- --host 0.0.0.0 --port 3000

# Отключение от сессии: нажмите Ctrl+A, затем D
# Возврат к сессии: screen -r urlshortener
```

## 🔒 Шаг 9: Настройка файрвола (если нужно)

Если порт 3000 закрыт файрволом:

```bash
# Для UFW (Ubuntu Firewall)
sudo ufw allow 3000/tcp
sudo ufw status

# Для iptables
sudo iptables -A INPUT -p tcp --dport 3000 -j ACCEPT
sudo iptables-save
```

## 📊 Шаг 10: Настройка базы данных Supabase

### 10.1 Создайте таблицы в Supabase

1. Откройте Supabase Dashboard → **SQL Editor**
2. Создайте новый запрос
3. Вставьте SQL для создания таблиц:

```sql
-- Таблица для коротких ссылок
CREATE TABLE IF NOT EXISTS urls (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    short_code VARCHAR(10) UNIQUE NOT NULL,
    original_url TEXT NOT NULL,
    custom_domain VARCHAR(255),
    user_id UUID REFERENCES auth.users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    clicks INTEGER DEFAULT 0,
    is_active BOOLEAN DEFAULT true
);

-- Таблица для статистики переходов
CREATE TABLE IF NOT EXISTS clicks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    url_id UUID REFERENCES urls(id) ON DELETE CASCADE,
    clicked_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    ip_address VARCHAR(45),
    user_agent TEXT,
    country VARCHAR(100),
    city VARCHAR(100),
    referer TEXT
);

-- Таблица для кастомных доменов
CREATE TABLE IF NOT EXISTS custom_domains (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES auth.users(id),
    domain VARCHAR(255) UNIQUE NOT NULL,
    is_verified BOOLEAN DEFAULT false,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- Таблица для управления контентом (CMS)
CREATE TABLE IF NOT EXISTS cms_content (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    key VARCHAR(100) UNIQUE NOT NULL,
    content JSONB NOT NULL,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_by UUID REFERENCES auth.users(id)
);

-- Включаем Row Level Security
ALTER TABLE urls ENABLE ROW LEVEL SECURITY;
ALTER TABLE clicks ENABLE ROW LEVEL SECURITY;
ALTER TABLE custom_domains ENABLE ROW LEVEL SECURITY;
ALTER TABLE cms_content ENABLE ROW LEVEL SECURITY;

-- Политики для urls
CREATE POLICY "Users can view all URLs" ON urls FOR SELECT USING (true);
CREATE POLICY "Users can create URLs" ON urls FOR INSERT WITH CHECK (
    auth.uid() = user_id OR user_id IS NULL
);
CREATE POLICY "Users can update own URLs" ON urls FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "Users can delete own URLs" ON urls FOR DELETE USING (auth.uid() = user_id);

-- Политики для clicks
CREATE POLICY "Anyone can insert clicks" ON clicks FOR INSERT WITH CHECK (true);
CREATE POLICY "Users can view clicks for own URLs" ON clicks FOR SELECT USING (
    EXISTS (
        SELECT 1 FROM urls WHERE urls.id = clicks.url_id AND urls.user_id = auth.uid()
    )
);

-- Политики для custom_domains
CREATE POLICY "Users can view own domains" ON custom_domains FOR SELECT USING (auth.uid() = user_id);
CREATE POLICY "Users can create own domains" ON custom_domains FOR INSERT WITH CHECK (auth.uid() = user_id);
CREATE POLICY "Users can update own domains" ON custom_domains FOR UPDATE USING (auth.uid() = user_id);
CREATE POLICY "Users can delete own domains" ON custom_domains FOR DELETE USING (auth.uid() = user_id);

-- Политики для cms_content
CREATE POLICY "Everyone can read CMS content" ON cms_content FOR SELECT USING (true);
CREATE POLICY "Only admins can modify CMS" ON cms_content FOR ALL USING (
    EXISTS (
        SELECT 1 FROM auth.users 
        WHERE id = auth.uid() 
        AND raw_user_meta_data->>'role' = 'admin'
    )
);

-- Индексы для производительности
CREATE INDEX idx_urls_short_code ON urls(short_code);
CREATE INDEX idx_urls_user_id ON urls(user_id);
CREATE INDEX idx_clicks_url_id ON clicks(url_id);
CREATE INDEX idx_clicks_clicked_at ON clicks(clicked_at);
CREATE INDEX idx_custom_domains_user_id ON custom_domains(user_id);
```

4. Нажмите **Run** (▶️)

## 🐛 Решение проблем

### Проблема: "Port 3000 already in use"

```bash
# Найти процесс на порту 3000
sudo lsof -i :3000

# Убить процесс (замените PID на номер из предыдущей команды)
sudo kill -9 PID
```

### Проблема: Сайт недоступен с внешнего IP

1. Проверьте файрвол:
   ```bash
   sudo ufw status
   ```

2. Проверьте, что приложение слушает 0.0.0.0:
   ```bash
   netstat -tlnp | grep 3000
   ```

3. Проверьте security groups вашего облачного провайдера (AWS, DigitalOcean и т.д.)

### Проблема: "Module not found" или ошибки импорта

```bash
# Удалите node_modules и переустановите
rm -rf node_modules package-lock.json
npm install
npm run build
```

### Проблема: Ошибки Supabase подключения

1. Проверьте `utils/supabase/info.tsx` - правильные ли данные
2. Проверьте в Supabase Dashboard → Settings → API - активен ли проект
3. Проверьте консоль браузера (F12) для деталей ошибки

## 👨‍💼 Создание админ пользователя

1. Зарегистрируйте пользователя через интерфейс сайта
2. Откройте Supabase Dashboard → **Authentication → Users**
3. Найдите вашего пользователя, скопируйте его ID
4. Откройте **SQL Editor** и выполните:

```sql
UPDATE auth.users 
SET raw_user_meta_data = jsonb_set(
    COALESCE(raw_user_meta_data, '{}'::jsonb),
    '{role}',
    '"admin"'
)
WHERE id = 'ваш-user-id';
```

5. Перезайдите на сайт - теперь у вас есть доступ к Admin Panel!
